# hook-memory

このツールは、ゲーム/アプリ上で以下を行うためのモニタです。
- HOOK（dylib -> class -> method）
- バイナリ監視（__TEXT ハッシュ監視）
- メモリ追跡（検索/編集/ロック/履歴）
- txtメモリパッチ（常時適用）
- オートクリック（マーカー自動タップ）
- テーマ色変更などの設定

## 起動方法
1. 画面上の赤い `星那` ボタンをタップ
2. `Monitor` メニューが開く

---

## メインメニュー
- `HOOK`
  - 読み込み済み dylib 一覧を表示
  - 対象を選んで class/method へ進む
- `バイナリ監視`
  - 変更検知時にアラート
- `メモリ追跡`
  - 値検索、絞り込み検索、値変更、ロック
  - 長押しで周辺アドレス表示
- `txtメモリパッチ`
  - txt定義を読み込み、チェック項目を常時書き換え
- `オートクリック`
  - マーカー位置へ順番タップ
  - レイアウト txt 保存/読込
- `ツール`
  - ログ、補助機能
- `設定`
  - 更新間隔、表示、テーマ色など

---

## txtメモリパッチの使い方
形式:
```txt
[名前]
base 0x00 5
base 0x04 6

[別名]
base 0x10 val
```

ルール:
- `[名前]` が1項目
- その下の `base ...` 行は同一項目内で複数可
- `val` は有効化時に手入力値を使用
- チェックした項目は 0.5秒間隔で適用（画面を閉じても継続）

Baseモード:
- Main（メイン実行バイナリ基準）
- Min（最小アドレス基準）
- Module（任意モジュール基準）

ファイル操作:
- `読み込み` で txt を選択
- `保存` で txt を保存

検索（アドレス取得 -> パッチ）
例:
```txt
search 0x100000000-0x200000000 100 & 90 & 80
get(2) 0x04 777
```

`val` 使用例:
```txt
search 0x100000000-0x200000000 100
get(1) 0x04 val
```

ルール:
- `&` で区切った回数だけ継続検索
- 範囲指定が無い場合は `0x1-0x200000000`
- 複数検索で範囲指定は最初の1回のみ
- `get(x)` の `x` はヒット上から何番目か（1始まり）

## メモリ追跡の使い方
対応型:
- `auto, float, double, uint8, uint16, uint32, uint64, int8, int16, int32, int64, text`

検索:
- 新規検索: 全リセットして検索
- 続けて検索: 前回ヒット結果から再検索
- 範囲未指定時は `0x1-0x200000000`

編集:
- 行タップ: 値変更
- ロック: 定期固定
- 長押し: 周辺アドレス/値を表示

差分表示:
- Base基準との差分を `0x` 16進で表示

## オートクリックの使い方
操作:
- `+` マーカー追加
- `-` 最大番号マーカー削除
- マーカータップで間隔(秒)を設定
- `Start` でループ開始/停止
- `Menu` でメインメニューへ戻る

実行仕様:
- マーカー番号の昇順で処理
- 各マーカーは個別間隔で発火
- 最小間隔ガードあり（取りこぼし低減）

### オートクリックの保存/読込
- `Save`: レイアウトを txt 保存（Files保存UI + ローカル保存）
- `Load`: txtを選択してレイアウト復元

保存形式:
```txt
# autoclick layout v1
# id,x,y,interval
1,120.000,300.000,0.5000
2,200.000,380.000,1.0000
```

ローカル保存先:
- `Documents/autoclick_layout.txt`

## テーマ変更
`設定 -> カラー設定` から以下を16進カラーで変更可能:
- 背景色
- 文字色
- 補助文字色
- 枠色

---

## 注意
- Private API を含むため、iOS/対象アプリで挙動差があります。
- 反応率が低い場合は、オートクリック間隔を少し長めに設定してください。
